def plot_confusion_matrix(y_true, y_pred, classes, normalize=False, title=None, cmap=plt.cm.Blues):
    if not title:
        if normalize:
            title = 'Matriz de confusin normalizada'
        else:
            title = 'Matriz de confusin sin normalizar'

    # Calcular la matriz de confusin
    cm = confusion_matrix(y_true, y_pred)
    
    # Normalizar la matriz de confusin si se solicita
    if normalize:
        cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]

    # Plot
    plt.figure(figsize=(8, 6))
    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    fmt = '.2f' if normalize else 'd'
    thresh = cm.max() / 2.
    for i in range(cm.shape[0]):
        for j in range(cm.shape[1]):
            plt.text(j, i, format(cm[i, j], fmt),
                     ha="center", va="center",
                     color="white" if cm[i, j] > thresh else "black")

    plt.ylabel('Etiqueta real')
    plt.xlabel('Etiqueta predicha')
    plt.tight_layout()
    plt.show()

def plot_ROC(y_true, y_prob):
    fpr, tpr, thresholds = roc_curve(y_true, y_prob)
    roc_auc = auc(fpr, tpr)

    plt.figure()
    plt.plot(fpr, tpr, color='darkorange', lw=2, label='ROC curve (area = %0.2f)' % roc_auc)
    plt.plot([0, 1], [0, 1], color='navy', lw=2, linestyle='--')
    plt.xlim([0.0, 1.0])
    plt.ylim([0.0, 1.05])
    plt.xlabel('Tasa de Falsos Positivos')
    plt.ylabel('Tasa de Verdaderos Positivos')
    plt.title('Curva ROC')
    plt.legend(loc="lower right")
    plt.show()

def train_and_evaluate_model(X_train, y_train, X_test, y_test, representation):
    # Crear el clasificador Random Forest
    classifier = RandomForestClassifier(n_estimators=100, random_state=42)

    # Entrenar el modelo
    classifier.fit(X_train, y_train)

    # Predecir las etiquetas de clase en el conjunto de prueba
    y_pred = classifier.predict(X_test)

    # Calcular la matriz de confusin
    cm = confusion_matrix(y_test, y_pred)

    # Calcular precision y recall
    precision = precision_score(y_test, y_pred, pos_label='spam')
    recall = recall_score(y_test, y_pred, pos_label='spam')

    # Calcular la puntuacin AUC y la curva ROC
    y_prob = classifier.predict_proba(X_test)[:, 1]

    # Convertir las etiquetas a valores binarios
    lb = LabelBinarizer()
    y_test_binary = lb.fit_transform(y_test)

    # Calcular la curva ROC y el rea bajo la curva (AUC)
    fpr, tpr, _ = roc_curve(y_test_binary, y_prob)
    auc = roc_auc_score(y_test_binary, y_prob)

    # Imprimir las mtricas
    print(f"Matriz de confusin ({representation}):")
    print(cm)
    print(f"Precisin ({representation}): {precision:.2f}")
    print(f"Recall ({representation}): {recall:.2f}")
    print(f"AUC ({representation}): {auc:.2f}")

    # Llamar a las funciones plot_confusion_matrix y plot_ROC
    plot_confusion_matrix(y_test, y_pred, classes=["ham", "spam"], normalize=True, title=f"Matriz de confusin ({representation})")
    plot_ROC(y_test_binary, y_prob)

1. Qu error es ms aceptable: dejar pasar un SMS de SPAM (falso negativo) o bloquear un SMS legtimo (falso positivo)? Justifique su respuesta.

En el caso de la deteccin de spam en SMS, generalmente es ms aceptable bloquear un SMS legtimo (falso positivo) que dejar pasar un SMS de spam (falso negativo). Esto se debe a que los usuarios pueden perder la confianza en el sistema si continan recibiendo mensajes de spam, lo que puede resultar en una experiencia de usuario negativa. Por otro lado, bloquear un SMS legtimo puede ser una molestia, pero es menos perjudicial que permitir que un mensaje de spam llegue al usuario final.

2. Compare los valores para cada modelo de representacin numrico. En base a la respuesta de la primera pregunta Qu modelo de representacin numrica produjo el mejor resultado, BoG o TF-IDF? Cul o cules son las razones por las que dicho modelo se comport de mejor manera?

En base a la respuesta a la primera pregunta, el modelo que produjo el mejor resultado sera el que tiene un menor nmero de falsos negativos, es decir, el que tiene un mayor recall. Si observamos los resultados proporcionados, el modelo TF-IDF tiene un recall ms alto en comparacin con los modelos BoG (n=1) y BoG (n=2). Por lo tanto, el modelo TF-IDF sera preferible en este caso, ya que minimiza la probabilidad de dejar pasar un SMS de spam (falso negativo), lo que resulta en una mejor experiencia para el usuario final.

En base a la exploracin de datos e ingeniera de caractersticas que realiz en el primer y este laboratorio, qu consejos le dara a un familiar que le solicita ayuda para detectar si un email o SMS es phishing o no? En qu caractersticas de una URL/email podra fijarse su familiar para ayudarlo a detectar un potencial phishing?

Algunos consejos para detectar si un email o SMS es phishing incluyen:

  • Verificar la autenticidad de los remitentes: Siempre revisar la direccin de correo electrnico o el nmero de telfono del remitente para asegurarse de que sea legtimo.
  • No hacer clic en enlaces sospechosos: Evitar hacer clic en enlaces incluidos en correos electrnicos o mensajes de texto que parezcan sospechosos o inesperados.
  • Revisar la gramtica y el formato: Prestar atencin a la gramtica y el formato del mensaje, ya que los correos electrnicos o mensajes de phishing suelen contener errores gramaticales o tener un formato inusual.

En cuanto a las caractersticas de una URL o email que podran indicar un posible phishing, su familiar debera fijarse en lo siguiente:

  • El dominio de la URL: Verificar si el dominio de la URL coincide con el sitio web esperado. Los sitios web de phishing a menudo utilizan dominios falsos o similares para engaar a los usuarios.
  • Solicitudes de informacin confidencial: Ser cauteloso con los correos electrnicos o mensajes que solicitan informacin confidencial, como contraseas, nmeros de cuenta bancaria o nmeros de tarjeta de crdito.
  • Urgencia o amenazas: Tener cuidado con los correos electrnicos o mensajes que inducen a la urgencia o amenazan con consecuencias negativas si no se toma una accin inmediata.

4. Si detectamos una URL o email/SMS de phishing, qu podemos hacer para detener su distribucin?

Algunas acciones que se pueden tomar para detener la distribucin de URLs o correos electrnicos/SMS de phishing incluyen:

  • Reportar el correo electrnico o mensaje de phishing a la compaa o institucin afectada para que puedan tomar medidas.
  • Bloquear la direccin de correo electrnico o nmero de telfono del remitente para evitar recibir ms mensajes de phishing.
  • Actualizar y mantener actualizado el software de seguridad, como los programas antivirus y antimalware, para detectar y bloquear mensajes de phishing.
  • Educar a los usuarios sobre cmo reconocer y evitar el phishing proporcionando capacitacin y concienciacin sobre seguridad en lnea.
